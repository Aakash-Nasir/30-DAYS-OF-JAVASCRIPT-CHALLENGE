Reason: let variables are block-scoped, meaning they are limited to the block ({ }) in which they are defined. They allow flexibility by allowing you to change the value of the variable within the same block scope without constraints once it's declared.
Reason: Unlike let, var is function-scoped, meaning it is visible and accessible tbroughout the entire function in which it is defined. This can sometimes lead to unexpected behavior due to hoisting, where variables are moved to the top of their function scope during compilation.
Reason: const is block-scoped like let, but it prevents reassignment of the variable itself after initialization. This is useful when you want to ensure that a variable doesn't change its reference or value accidentally, promoting immutability for certain types of data structures.
Why const is Considered Immutable (Reference-wise): Reference Stability: The immutability associated with const primarily refers to the stability of the reference itself. Once a const variable is assigned a value, the variable cannot be reassigned to refer to a different value. This helps prevent accidental reassignments and promotes code clarity and predictability. Intent and Readability: Using const makes your code more readable by signaling to other developers that the variable should not be reassigned. It also helps prevent unintended side effects that can occur when variables are redefined unexpectedly
